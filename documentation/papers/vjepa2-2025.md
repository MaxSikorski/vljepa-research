## 1. Overview

V-JEPA 2 is the successor to V-JEPA and represents the most capable model in the JEPA lineage for video understanding. It scales the JEPA approach to 1.2 billion parameters and introduces block-causal attention with autoregressive feature prediction, enabling not just video understanding but also future prediction and robotic planning.

### Key Contributions

1. A 1.2B parameter ViT trained with JEPA on 1M+ hours of video.
2. Block-causal attention: a hybrid of bidirectional and causal attention that enables both understanding and prediction.
3. Autoregressive feature prediction in latent space.
4. V-JEPA 2-AC: an action-conditioned world model variant.
5. Zero-shot robotic control via Model Predictive Control (MPC).
6. State-of-the-art on video understanding benchmarks AND physics reasoning benchmarks.

## 2. Architecture

### 2.1 Core: 1.2B Parameter ViT

The backbone is a Vision Transformer at massive scale:

| Property | Value |
|----------|-------|
| Architecture | ViT-g (custom variant) |
| Parameters | 1.2 Billion |
| Embedding dimension | 1536 |
| Attention heads | 24 |
| Transformer layers | 40 |
| Patch size | 16x16 spatial, 2 frames temporal |
| Input resolution | 224x224 spatial, variable temporal |

The model processes video as a sequence of spatiotemporal patch tokens. Each token covers a 16x16 spatial region across 2 consecutive frames.

### 2.2 Block-Causal Attention

This is the key architectural innovation of V-JEPA 2.

**Problem**: Standard JEPA uses bidirectional attention (I-JEPA, V-JEPA). This is great for understanding but cannot do temporal prediction -- you cannot attend to future frames when generating predictions autoregressively.

**Solution**: Block-causal attention partitions the token sequence into temporal blocks and applies:
- **Bidirectional attention within each temporal block**: Tokens from the same time step can attend to each other freely. This preserves the spatial understanding capability.
- **Causal attention across temporal blocks**: Each temporal block can only attend to itself and previous blocks, never future blocks. This enables autoregressive prediction.

```
Time:       t=1        t=2        t=3        t=4
Tokens:   [●●●●]    [●●●●]    [●●●●]    [●●●●]

Attention pattern:
  t=1 tokens: attend to t=1 only          (bidirectional within block)
  t=2 tokens: attend to t=1, t=2          (causal across blocks, bidirectional within)
  t=3 tokens: attend to t=1, t=2, t=3     (causal across blocks, bidirectional within)
  t=4 tokens: attend to t=1, t=2, t=3, t=4
```

**Why this matters**:
- For understanding tasks (classification, retrieval): Process all frames with effectively bidirectional attention by providing all frames as input. Each block sees all previous context.
- For prediction tasks: Given frames t=1..k, predict the features of frame t=k+1 without seeing it. The causal mask ensures no information leakage.
- Single architecture serves both purposes without modification.

### 2.3 Autoregressive Feature Prediction

Building on block-causal attention, V-JEPA 2 can generate future frame features autoregressively:

```
Given:   f(t=1), f(t=2), f(t=3)     [encoded features of observed frames]
Predict: f(t=4)_hat                   [predicted features of next frame]

Then:    f(t=1), f(t=2), f(t=3), f(t=4)_hat
Predict: f(t=5)_hat                   [predicted features of t+2]

Continue for arbitrary horizon...
```

Critical distinction from generative video models: V-JEPA 2 predicts abstract features, NOT pixels. The predictions represent semantic content (what objects are where, what is happening) without committing to specific visual details.

## 3. Training

### 3.1 VideoMix22M Dataset

| Property | Value |
|----------|-------|
| Name | VideoMix22M |
| Total videos | ~22 million clips |
| Total duration | 1 million+ hours |
| Sources | Multiple video datasets (details in paper) |
| Sampling | Variable frame rate, uniform temporal sampling |
| Resolution | 224x224 (training) |

This is a massive step up from V-JEPA's training data and is critical for the model's strong performance.

### 3.2 Training Procedure

**Masking strategy**: Extends the I-JEPA multi-block masking to spatiotemporal volumes:
- Target regions are 3D blocks (spatial x spatial x temporal).
- Context is the complement of the target blocks.
- The model predicts latent features of masked spatiotemporal regions from the visible context.

**Target encoder**: EMA of the context encoder, following the I-JEPA precedent.

**Objective**: L2 prediction loss in latent space (same as I-JEPA / V-JEPA).

### 3.3 Cooldown Phase

V-JEPA 2 introduces a "cooldown" phase at the end of training:
- Reduces learning rate to near-zero.
- Trains on higher-quality data subset.
- Stabilizes representations for downstream use.
- Analogous to the "annealing" phase in LLM training.

This cooldown is important for getting the best downstream performance. Without it, the representations can be slightly noisier.

## 4. V-JEPA 2-AC: Action-Conditioned Variant

### 4.1 Concept

V-JEPA 2-AC extends the base model to condition predictions on robot actions, transforming it from a passive world model into an active one suitable for robot control.

```
Standard V-JEPA 2:
  visual features (t=1..k) --> predict visual features (t=k+1)

V-JEPA 2-AC:
  visual features (t=1..k) + actions (a=1..k) --> predict visual features (t=k+1)
```

### 4.2 Droid Dataset

| Property | Value |
|----------|-------|
| Name | Droid |
| Type | Robot manipulation demonstrations |
| Duration | 62 hours of robot manipulation data |
| Robot | Franka Emika Panda arm |
| Actions | 7-DOF joint velocities + gripper |
| Scenes | Diverse tabletop manipulation scenarios |

62 hours is remarkably small. The key insight is that the base V-JEPA 2 model already understands physics, objects, and spatial relationships from its massive video pretraining. The Droid fine-tuning only needs to teach the mapping between actions and their visual consequences.

### 4.3 Training V-JEPA 2-AC

- Start from the pretrained V-JEPA 2 backbone.
- Add action conditioning: robot action vectors are projected and added to the token representations at each time step.
- Fine-tune on Droid data with the same JEPA objective but now conditioned on actions.
- The model learns: "Given what I see and what the robot does, what will the scene look like next?"

## 5. Robotics Results: Zero-Shot Franka Arm Control

### 5.1 Model Predictive Control (MPC)

V-JEPA 2-AC enables zero-shot robot control through MPC:

```
1. Observe current scene --> encode to features f(t)
2. Define goal --> encode goal image/state to features f_goal
3. For many candidate action sequences [a1, a2, ..., aH]:
     a. Simulate trajectory through V-JEPA 2-AC:
        f(t) + a1 --> f(t+1)_hat
        f(t+1)_hat + a2 --> f(t+2)_hat
        ...
     b. Compute cost = distance(f(t+H)_hat, f_goal) in latent space
4. Execute the first action from the lowest-cost sequence
5. Re-observe and repeat (receding horizon control)
```

### 5.2 Key Results

- **Zero-shot control**: No task-specific training. The world model learned from passive video + Droid data is sufficient.
- **Franka Panda arm**: Successfully controls a real 7-DOF robot arm.
- **Tasks**: Object reaching, pushing, basic manipulation.
- **Planning horizon**: Multiple steps into the future.
- **Replanning**: Continuous replanning at each time step handles disturbances.

### 5.3 Why This Matters

This is a direct realization of LeCun's 2022 vision:
- World model (V-JEPA 2-AC) predicts consequences of actions in latent space.
- Cost module (distance to goal features) evaluates predicted states.
- Actor (MPC optimizer) plans action sequences through the world model.
- No reinforcement learning needed -- pure planning with a learned world model.

## 6. Performance Benchmarks

### 6.1 Video Classification

| Benchmark | V-JEPA 2 | Notes |
|-----------|----------|-------|
| Something-Something v2 (SSv2) | **77.3%** | Strong temporal reasoning required |
| Kinetics-400 | Competitive SOTA | Appearance-heavy benchmark |

SSv2 at 77.3% is particularly significant because SSv2 requires understanding temporal dynamics (e.g., "pushing something from left to right" vs. "pushing something from right to left"). High SSv2 performance indicates genuine temporal understanding, not just appearance matching.

### 6.2 Video Retrieval

| Benchmark | Metric | V-JEPA 2 |
|-----------|--------|----------|
| Epic Kitchens | R@5 | **39.7** |

Epic Kitchens is a challenging egocentric video dataset with fine-grained action classes.

### 6.3 Physics Reasoning (New Benchmarks)

V-JEPA 2 introduces evaluation on physics reasoning tasks:
- Predicting physical outcomes (will the tower fall?).
- Understanding forces and trajectories.
- Reasoning about object interactions.

These benchmarks test whether the model has learned an implicit physical world model, going beyond standard action recognition.

### 6.4 Comparison Context

V-JEPA 2's results should be understood in the context of:
- It uses NO language supervision (unlike CLIP-based models).
- It uses NO labeled data during pretraining.
- It learns purely from watching video.
- The representations transfer to robotics without task-specific RL.

## 7. Relationship to Other Models in the Lineage

### V-JEPA 2 vs. V-JEPA (v1)

| Aspect | V-JEPA | V-JEPA 2 |
|--------|--------|----------|
| Parameters | 300M (ViT-L) | 1.2B (ViT-g) |
| Attention | Bidirectional only | Block-causal (bidirectional + causal) |
| Prediction | Masked prediction only | Masked + autoregressive |
| Training data | ~2M clips | 22M clips (1M+ hours) |
| Cooldown | No | Yes |
| Action conditioning | No | Yes (V-JEPA 2-AC) |
| Robotics | No | Yes (zero-shot MPC) |
| Physics reasoning | Not evaluated | Yes |

### V-JEPA 2 vs. I-JEPA

| Aspect | I-JEPA | V-JEPA 2 |
|--------|--------|----------|
| Modality | Images | Video |
| Masking | 2D spatial blocks | 3D spatiotemporal blocks |
| Temporal modeling | None | Block-causal + autoregressive |
| Scale | Up to 632M | 1.2B |
| Applications | Image classification | Video understanding + robotics |

### V-JEPA 2's Role in VL-JEPA

VL-JEPA uses a **frozen V-JEPA 2 ViT-L** (304M parameters, not the full 1.2B ViT-g) as its visual encoder (X-Encoder). The V-JEPA 2 backbone provides:
- Rich spatiotemporal features.
- Semantic understanding of video content.
- No need for visual fine-tuning -- the features are good enough to use frozen.

Note: VL-JEPA uses the ViT-L variant (304M), not the full 1.2B ViT-g. This is likely for compute efficiency and because the ViT-L already provides sufficiently good features for the language integration task.

## 8. Technical Details Worth Noting

### Positional Embeddings

- 3D sinusoidal positional embeddings (spatial x spatial x temporal).
- Interpolated for different resolutions and video lengths.

### Patch Embedding

- Spatial: 16x16 pixel patches.
- Temporal: 2 consecutive frames per token.
- For a 16-frame video at 224x224: 14 x 14 x 8 = 1,568 tokens per video.

### Memory and Compute

The 1.2B model is expensive:
- Training on VideoMix22M required significant GPU resources (specifics in paper).
- Inference: Processing a single video requires ~1,568 tokens through a 40-layer, 1536-dim transformer.
- Batch processing and efficient attention implementations are necessary.

### Block-Causal Implementation

The block-causal attention mask can be implemented as:
```python
# Pseudocode
def block_causal_mask(num_blocks, tokens_per_block):
    mask = torch.zeros(total_tokens, total_tokens)
    for i in range(num_blocks):
        for j in range(num_blocks):
            if j <= i:  # causal across blocks
                # Allow full attention (bidirectional within and to past blocks)
                mask[i*tpb:(i+1)*tpb, j*tpb:(j+1)*tpb] = 1
    return mask
```

## 9. Open Questions and Limitations

1. **Scale gap for VL-JEPA**: Why does VL-JEPA use the 304M ViT-L rather than the 1.2B ViT-g? Is it purely compute, or does the larger model not help for language integration?

2. **Cooldown data selection**: What data is used in the cooldown phase, and how much does it matter?

3. **Droid data size**: 62 hours is tiny. What is the minimum data needed for effective action conditioning? Can this scale to more complex robots?

4. **Planning horizon**: How far into the future can V-JEPA 2-AC reliably predict? Error accumulation in autoregressive prediction is a known challenge.

5. **Multi-task robotics**: Current results are on relatively simple tasks. Can MPC with V-JEPA 2-AC handle complex, multi-step manipulation?

6. **Real-time inference**: Can the 1.2B model run fast enough for real-time robot control, or is the ViT-L variant necessary for practical deployment?

## 10. Key Takeaways for Our Lab

1. **Block-causal attention is a unifying design**: A single architecture handles both understanding and prediction. This pattern may be useful for our own experiments.

2. **Autoregressive feature prediction is the bridge to planning**: The ability to roll out future states in latent space is what enables robotics applications.

3. **Massive pretraining transfers**: 1M+ hours of video pretraining means the model has seen enough physics to enable zero-shot control with only 62 hours of robot data.

4. **V-JEPA 2 ViT-L is the backbone for VL-JEPA**: Understanding V-JEPA 2 is prerequisite for understanding VL-JEPA's visual encoder.
